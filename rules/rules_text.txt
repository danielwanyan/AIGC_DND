# 商品介绍详细度评估判定规则
# Description Detail Evaluation Rules

## 版本信息

- 当前版本：2026-06-01-v1
- 上次更新：2026-06-01
- 更新内容：初始版本，基于 ASR 的商品介绍详细度评估规则

---

## 规则索引

| 编号 | 规则名称 | 说明 |
|------|---------|------|
| DND01 | 介绍不详细 | ASR 中商品介绍内容不足 2 类，或内容过于笼统 |

---

## 适用范围

本规则适用于 **所有地区所有商品类目**（v1.0 版本，暂不分类目）。

**重要说明**：
- v1.0 版本只基于 ASR 判断，从严筛选
- 后续版本可能加入 OCR、视觉、商品类目豁免等规则
- 所有内容都需要分析，不会跳过

---

## 输入信息

### 文本信息

| 字段 | 说明 |
|------|------|
| ASR | 视频语音转文本（唯一判断依据） |
| OCR | 视频画面中识别出的文字（v1.0 忽略，保留供后续使用） |
| product_category | 挂车商品的三级类目名称（v1.0 忽略，保留供后续使用） |
| item_id | 商品 ID |
| aigc_video_id | 视频 ID |

---

## 评估标准（全商品适用，不分类目，从严筛选）

### ✅ OK（介绍足够详细）

ASR 中包含以下内容中的**至少 2 项**，且每项都有**具体描述**：

| 类别 | 说明 | 示例 |
|------|------|------|
| 1. 功能/功效介绍 | 商品的功能、用途、功效等 | "这款耳机支持主动降噪，降噪深度可达 40dB"、"这款面霜具有保湿功效，能够持续 24 小时锁水" |
| 2. 特点/优势说明 | 商品的特点、优势、卖点等 | "采用最新款 A17 处理器，性能提升 30%"、"纯天然植物成分，无刺激" |
| 3. 使用方法/适用场景 | 商品的使用方法、适用场景、适用人群等 | "每天早晚洁面后使用，取适量涂抹于面部"、"适合运动时佩戴，防水防汗" |
| 4. 参数/规格说明 | 商品的参数、规格、尺寸、容量等 | "6.7 英寸 OLED 屏幕，分辨率 2796×1290"、"5000mAh 大容量电池" |
| 5. 材质/成分介绍 | 商品的材质、成分、原料等 | "100% 纯棉材质，柔软透气"、"含有维生素 C 和烟酰胺成分" |

**关键要求**：
- 每项介绍必须有**具体描述**，不能只有笼统的说法
- 例如：只说"功能强大"不算有功能介绍，必须说明具体是什么功能
- 例如：只说"材质好"不算有材质介绍，必须说明具体是什么材质

### ❌ Description is not detailed（介绍不够详细）

满足以下任意一条即判定为不详细：

| 情况 | 说明 |
|------|------|
| ASR 为空 | 没有语音内容，或只有语气词、背景音乐、笑声等 |
| 只有商品名称 | ASR 只有简单的商品名称，没有任何介绍内容 |
| 只有空洞词汇 | ASR 只有"好看"、"推荐"、"买它"、"性价比高"、"很棒"等空洞词汇，没有具体介绍 |
| 符合项 < 2 | ASR 介绍内容少于上述 2 类 |
| 内容过于笼统 | ASR 有介绍但过于笼统，没有具体描述（如只说"功能强大"但不说具体是什么功能） |

---

## 判定流程

```
Step 1: 检查 ASR 内容
    ├─ ASR 为空或只有语气词/背景音乐 → ❌ 不详细
    ├─ ASR 只有商品名称或空洞词汇 → ❌ 不详细
    └─ ASR 有实质内容 → Step 2

Step 2: 识别 5 类介绍内容
    ├─ 逐类检查 ASR 中是否有该类介绍
    ├─ 每类介绍必须有具体描述，不能笼统
    └─ 有介绍必须引用 ASR 原文作为证据 → Step 3

Step 3: 统计符合项数量
    ├─ 符合项 ≥ 2 → ✅ OK
    └─ 符合项 < 2 → ❌ 不详细
```

## 豁免规则（v1.0 暂不启用，预留位置）

**重要说明**：v1.0 版本暂不启用任何豁免规则，所有商品统一标准。

后续版本可能加入的豁免规则类型（预留）：

| 豁免类型 | 说明 | 状态 |
|---------|------|------|
| 普通商品豁免 | 某些普通商品（如垃圾袋、纸巾）可能不需要详细介绍 | 预留 |
| 特定类目豁免 | 某些类目（如盲盒、首饰）可能适用不同标准 | 预留 |
| 短视频豁免 | 视频时长过短（如 < 15秒）可能豁免 | 预留 |

## 防误判规则（重要！必须遵守）

### 规则 FP-01：具体描述 vs 笼统描述

**问题场景**：ASR 中提到了某类内容，但过于笼统，没有具体信息。

**判定规则**：
- ✅ **有具体描述**才算符合：必须说明具体是什么、怎么样
- ❌ **笼统描述**不算符合：如"功能强大"、"材质好"、"性价比高"
- ❌ **空洞词汇**不算符合：如"好看"、"推荐"、"买它"、"很棒"

**示例**：
- "这款耳机支持主动降噪" → ✅ 有具体功能介绍
- "这款耳机功能很强" → ❌ 笼统描述，不算符合

---

### 规则 FP-02：从严判定边界情况

**问题场景**：某些介绍内容介于有和无之间，难以明确判定。

**判定规则**：
- ✅ **从严判定**：不确定或模棱两可的情况，一律判定为"无"
- ✅ **宁枉勿纵**：v1.0 目标是从严筛选，宁可误判不详细，也不漏判
- ❌ 不得将模糊表述判定为"有"

**示例**：
- "这款耳机采用了先进的技术" → ❌ 模糊表述，判定为"无"
- "这款耳机采用了蓝牙 5.3 技术" → ✅ 具体，判定为"有"

---

### 规则 FP-03：证据引用必须准确

**问题场景**：判定"有"某类介绍时，引用的 ASR 原文不准确或不完整。

**判定规则**：
- ✅ **逐字引用**：引用的内容必须是 ASR 中真实存在的原文
- ✅ **完整引用**：引用的内容必须足够完整，能够支撑判定
- ❌ 不得 paraphrase（意译），必须引用原文
- ❌ 不得编造不存在的内容

---

## 判定逻辑总结

| 情况 | 结果 | 说明 |
|------|------|------|
| ASR 为空或只有语气词 | ❌ 不详细 | 没有实质内容 |
| ASR 只有商品名称或空洞词汇 | ❌ 不详细 | 没有具体介绍 |
| ASR 有实质内容但符合项 < 2 | ❌ 不详细 | 介绍内容不足 |
| ASR 有实质内容且符合项 ≥ 2 | ✅ OK | 介绍足够详细 |

## 判定示例

### 示例 1：✅ OK（3 项符合）

**ASR 内容**："这款耳机支持主动降噪，降噪深度可达 40dB，采用最新款蓝牙 5.3 芯片，续航长达 30 小时，适合运动时佩戴，防水防汗。"

**判定**：✅ OK
**符合项**：
- 功能/功效介绍：有 - "支持主动降噪，降噪深度可达 40dB"
- 特点/优势说明：有 - "采用最新款蓝牙 5.3 芯片，续航长达 30 小时"
- 使用方法/适用场景：有 - "适合运动时佩戴，防水防汗"
- 符合项数量：3/5

---

### 示例 2：❌ 不详细（只有空洞词汇）

**ASR 内容**："这款耳机真的很好用，推荐大家购买，性价比超高！"

**判定**：❌ Description is not detailed
**符合项**：
- 功能/功效介绍：无
- 特点/优势说明：无
- 使用方法/适用场景：无
- 参数/规格说明：无
- 材质/成分介绍：无
- 符合项数量：0/5
**理由**：ASR 中只有"很好用"、"推荐"、"性价比超高"等空洞词汇，没有任何具体的商品介绍内容。

---

### 示例 3：❌ 不详细（只有 1 项符合）

**ASR 内容**："这款耳机采用蓝牙 5.3 芯片。"

**判定**：❌ Description is not detailed
**符合项**：
- 功能/功效介绍：无
- 特点/优势说明：有 - "采用蓝牙 5.3 芯片"
- 使用方法/适用场景：无
- 参数/规格说明：无
- 材质/成分介绍：无
- 符合项数量：1/5
**理由**：ASR 中只有 1 项介绍内容（特点/优势），不足 2 项，判定为不详细。

---

### 示例 4：✅ OK（刚好 2 项符合）

**ASR 内容**："这款耳机支持主动降噪，采用蓝牙 5.3 芯片。"

**判定**：✅ OK
**符合项**：
- 功能/功效介绍：有 - "支持主动降噪"
- 特点/优势说明：有 - "采用蓝牙 5.3 芯片"
- 使用方法/适用场景：无
- 参数/规格说明：无
- 材质/成分介绍：无
- 符合项数量：2/5
**理由**：ASR 中有 2 项具体介绍内容（功能/功效、特点/优势），符合要求。

---

### 示例 5：❌ 不详细（内容过于笼统）

**ASR 内容**："这款耳机功能强大，音质很好，性价比超高。"

**判定**：❌ Description is not detailed
**符合项**：
- 功能/功效介绍：无（"功能强大"过于笼统，没有具体说明是什么功能）
- 特点/优势说明：无（"音质很好"过于笼统，没有具体说明）
- 使用方法/适用场景：无
- 参数/规格说明：无
- 材质/成分介绍：无
- 符合项数量：0/5
**理由**：ASR 中的"功能强大"、"音质很好"都是笼统描述，没有具体说明是什么功能、音质怎么好，不符合"具体描述"要求（FP-01）。
